# ADR-015: 原地升级 Schema 而非新建文件

## 状态
PROPOSED

## 背景
State Schema 需要从 v2.0.0 升级到 v2.1.0，新增 `childrens` 数组和 `depth` 字段以支持树形结构。

可选方案：
1. 新建 `schema-v2.1.0.ts` 文件
2. 在 `schema-v2.0.0.ts` 中原地扩展

## 决策
在 `schema-v2.0.0.ts` 中原地扩展，而非新建文件。

新增内容：
- `ChildFeatureInfo` 接口
- `StateV2_1_0` 接口
- `adaptToV2_1_0()` 适配函数
- `validateStateV2_1_0()` 验证函数

## 后果

### 正面
- v2.1.0 是 v2.0.0 的超集，新增字段都有默认值（`childrens: []`, `depth: 1`）
- 减少模块数量，降低维护成本
- 向后兼容：旧版读取逻辑不受影响
- 版本字段使用联合类型 `'2.0.0' | '2.1.0'`

### 负面
- 需要确保 `adaptToV2_1_0()` 函数被所有读取 state.json 的地方调用
- 文件名与版本号不匹配（schema-v2.0.0.ts 包含 v2.1.0 定义）

### 缓解措施
- 在文件顶部添加注释说明已扩展至 v2.1.0
- 所有读取 state.json 的地方统一调用适配函数

## 相关决策
- ADR-001: 完全分布式存储 → 分布式 state.json 仍是最终目标
- ADR-009: 混合模式依赖检查 → v2.1.0 扩展依赖检查为跨子树模式
