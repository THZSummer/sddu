# ADR-016: TreeScanner 为纯模块

## 状态
PROPOSED

## 背景
TreeScanner 需要被 AutoUpdater、DependencyChecker、StateMachine 等多个模块使用。如果 TreeScanner 导入状态管理模块，可能产生循环依赖。

## 决策
TreeScanner 不导入任何状态管理模块，仅提供纯扫描函数。

输入：根目录路径  
输出：`ScanResult { nodes: FeatureTreeNode[], flatMap: Map<string, FeatureTreeNode> }`

## 后果

### 正面
- 避免循环依赖（tree-scanner → auto-updater → tree-scanner）
- 可独立测试，无需 mock 状态管理模块
- 符合单一职责原则
- 调用方可以根据需要组合使用扫描结果

### 负面
- 状态更新逻辑由调用方自行处理
- TreeScanner 的输出为原始数据结构，不含业务逻辑
- 需要额外的 parent-state-manager.ts 处理业务逻辑

### 缓解措施
- 通过 `ScanResult.flatMap` 提供 O(1) 路径查找能力
- parent-state-manager.ts 负责将扫描结果写入 state.json
