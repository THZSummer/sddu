# ADR-017: 完全分布式状态管理

## 状态
PROPOSED

## 背景
当前架构存在状态存储不一致问题：
- **StateMachine** 使用集中式 `.sdd/state.json`（包含所有 Feature 的状态）
- **DependencyChecker** 使用分布式 `<featureId>/state.json`（每个 Feature 独立文件）
- 两者可能不同步

## 决策
v2.4.0 将 StateMachine 彻底改为分布式，删除集中式 `.sdd/state.json`，所有模块统一依赖分布式 state.json 文件。

新增 `StateLoader` 类作为分布式状态加载器，替代集中式 `.sdd/state.json` 的读写逻辑。

## 后果

### 正面
- 架构统一，消除集中式 vs 分布式的不一致
- 树形结构天然适配分布式模型
- 零历史包袱，为后续版本奠定干净基础
- 符合 ADR-001 完全分布式存储的设计原则

### 负面
- 改动范围较大，需要重构 StateMachine 的 load/save 逻辑
- 需要新增 StateLoader、TreeStateValidator 等模块

### 风险缓解
- SDDU 处于内部使用阶段，未正式发布，无外部用户影响
- 充分单元测试确保重构质量
