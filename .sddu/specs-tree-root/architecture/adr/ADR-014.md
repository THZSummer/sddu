## 1. 背景

### 1.1 问题陈述

当前 SDD 插件的安装流程需要分别复制多个目录：

**现状** (`install.sh`):
```bash
# 分别复制多个目录
cp -r "$SCRIPT_DIR/dist/"* "$PLUGIN_DEST/"
cp -r "$SCRIPT_DIR/templates/agents/" "$PLUGIN_DEST/agents/"
```

**问题**:
1. **安装逻辑复杂**: 需要分别复制多个目录
2. **缺乏原子性**: 部分复制成功、部分失败会导致不一致
3. **分发格式缺失**: 没有打包文件便于 npm 发布或 GitHub Release
4. **测试困难**: 难以验证完整插件包内容

### 1.2 用户需求

| 需求 | 说明 | 优先级 |
|------|------|--------|
| 简化安装流程 | 安装时只需一步复制 | P1 |
| 清晰边界 | 插件包与开发文件分离 | P1 |
| 支持分发 | 便于 npm/GitHub Release 发布 | P2 |
| 原子性保证 | 要么全部成功，要么失败 | P1 |

### 1.3 决策范围

本 ADR 决定：
- 创建打包脚本生成 `dist/sdd/` 目录
- 生成 `dist/sdd.zip` 压缩文件
- 修改安装脚本优先使用打包结果
- 保持向后兼容（降级模式）

---

## 2. 决策

### 2.1 方案选择

我们决定采用 **打包分发** 方案：

**创建 `scripts/package.cjs` 打包脚本，生成 `dist/sdd/` 和 `dist/sdd.zip`**。

### 2.2 设计原则

1. **原子性**: 打包结果要么完整，要么失败
2. **清晰边界**: `dist/sdd/` 包含完整插件包，不含开发文件
3. **降级兼容**: 安装脚本支持多种来源（zip > sdd/ > dist/）
4. **便于分发**: zip 文件便于 npm/GitHub Release 发布

### 2.3 目录结构

### 2.4 实现细节

#### 2.4.1 打包脚本

**文件**: `scripts/package.cjs`

**依赖**:
```json
{
  "devDependencies": {
    "fs-extra": "^11.0.0",
    "archiver": "^6.0.0"
  }
}
```

**代码**:
```javascript
const fs = require('fs-extra');
const archiver = require('archiver');
const path = require('path');

const DIST_DIR = path.join(__dirname, '..', 'dist');
const SDD_DIR = path.join(DIST_DIR, 'sdd');
const ZIP_FILE = path.join(DIST_DIR, 'sdd.zip');

async function package() {
  console.log('📦 Starting package process...');
  
  // 1. 清理 dist/sdd/
  await fs.remove(SDD_DIR);
  await fs.mkdirp(SDD_DIR);
  console.log('✅ Cleaned dist/sdd/');
  
  // 2. 复制文件（排除 templates/ 和 sdd/）
  const items = await fs.readdir(DIST_DIR);
  for (const item of items) {
    if (item !== 'templates' && item !== 'sdd' && item !== 'sdd.zip') {
      await fs.copy(
        path.join(DIST_DIR, item),
        path.join(SDD_DIR, item)
      );
    }
  }
  console.log('✅ Copied core files');
  
  // 3. 复制 agents (从 templates/agents/)
  const templatesAgentsDir = path.join(DIST_DIR, 'templates', 'agents');
  const sddAgentsDir = path.join(SDD_DIR, 'agents');
  
  if (await fs.pathExists(templatesAgentsDir)) {
    await fs.copy(templatesAgentsDir, sddAgentsDir);
    console.log('✅ Copied agents to dist/sdd/agents/');
  } else {
    console.warn('⚠️  templates/agents/ not found, skipping');
  }
  
  // 4. 复制 package.json
  await fs.copy(
    path.join(__dirname, '..', 'package.json'),
    path.join(SDD_DIR, 'package.json')
  );
  console.log('✅ Copied package.json');
  
  // 5. 打包 zip
  await createZip(SDD_DIR, ZIP_FILE);
  console.log('✅ Created sdd.zip');
  
  // 6. 输出统计
  const stats = await fs.stat(ZIP_FILE);
  console.log(`📊 Package complete: ${ZIP_FILE} (${(stats.size / 1024).toFixed(2)} KB)`);
}

async function createZip(source, dest) {
  return new Promise((resolve, reject) => {
    const output = fs.createWriteStream(dest);
    const archive = archiver('zip', { zlib: { level: 9 } });
    
    output.on('close', resolve);
    archive.on('error', reject);
    archive.pipe(output);
    archive.directory(source, 'sdd');
    archive.finalize();
  });
}

package().catch((error) => {
  console.error('❌ Package failed:', error);
  process.exit(1);
});
```

```json
{

  },
  "devDependencies": {
    "fs-extra": "^11.0.0",
```

#### 2.4.3 安装脚本修改

**install.sh (片段)**:
```bash
# 优先使用打包后的插件包
if [ -f "$SCRIPT_DIR/dist/sdd.zip" ]; then
    echo "📦 解压插件包：dist/sdd.zip"
    unzip -o "$SCRIPT_DIR/dist/sdd.zip" -d "$TARGET_DIR/.opencode/plugins/"
elif [ -d "$SCRIPT_DIR/dist/sdd" ]; then
    echo "📦 使用打包后的插件包：dist/sdd/"
    cp -r "$SCRIPT_DIR/dist/sdd/"* "$PLUGIN_DEST/"
else
    echo "⚠️  未找到打包目录，使用降级模式：dist/"
    cp -r "$SCRIPT_DIR/dist/"* "$PLUGIN_DEST/"
fi
```

**install.ps1 (片段)**:
```powershell
# 优先使用打包后的插件包
if (Test-Path "$SCRIPT_DIR\dist\sdd.zip") {
    Write-Host "📦 解压插件包：dist\sdd.zip"
    Expand-Archive -Path "$SCRIPT_DIR\dist\sdd.zip" -DestinationPath "$TARGET_DIR\.opencode\plugins\" -Force
} elseif (Test-Path "$SCRIPT_DIR\dist\sdd") {
    Write-Host "📦 使用打包后的插件包：dist\sdd/"
    Copy-Item -Path "$SCRIPT_DIR\dist\sdd\*" -Destination "$PLUGIN_DEST\" -Recurse -Force
} else {
    Write-Host "⚠️  未找到打包目录，使用降级模式：dist/"
    Copy-Item -Path "$SCRIPT_DIR\dist\*" -Destination "$PLUGIN_DEST\" -Recurse -Force
}
```

### 2.5 安装流程（更新后）

```
1. 检查源码 (package.json)
2. 构建到 dist/ (tsc + build-agents.cjs)
3. 打包到 dist/sdd/ (package.cjs)
4. 创建 .opencode/plugins/sdd/ 目录
5. 复制/解压 dist/sdd/ 到 .opencode/plugins/sdd/
   - 优先使用 dist/sdd.zip 解压
   - 降级使用 dist/sdd/ 目录复制
   - 最后降级使用 dist/ 目录复制（向后兼容）
6. 版本检测和更新提示
7. opencode.json 智能合并
8. 初始化 .sdd/ 目录
```

## 3. 后果

### 3.1 正面影响

| 影响 | 说明 |
|------|------|
| ✅ 简化安装逻辑 | 一步复制完整插件包 |
| ✅ 原子性保证 | 要么全部成功，要么失败 |
| ✅ 清晰边界 | 插件包与开发文件分离 |
| ✅ 便于分发 | zip 文件用于 npm/GitHub Release |
| ✅ 便于测试 | 可直接测试 dist/sdd/ 内容 |

### 3.2 负面影响

| 影响 | 说明 | 缓解措施 |
|------|------|----------|
| ⚠️ 增加构建步骤 | 需要运行 package 脚本 | 集成到 CI/CD |
| ⚠️ 增加依赖 | 需要 fs-extra 和 archiver | 作为 devDependencies |
| ⚠️ 磁盘空间增加 | dist/sdd/ 和 dist/sdd.zip | 清理时删除 |

### 3.3 技术影响

- **编译时间**: 增加打包步骤（约 2-3 秒）
- **运行时性能**: 无影响
- **包大小**: dist/sdd.zip 约 50-100KB（压缩后）
- **向后兼容**: 完全兼容（降级模式）

## 4. 替代方案

### 方案 A: 使用 npm pack（已否决）

**描述**: 使用 `npm pack` 生成 tarball。

**优点**:
- 标准 npm 工具
- 无需额外依赖

**缺点**:
- 生成 .tgz 格式
- 需要 npm 环境
- 目录结构不可控

**否决理由**: 不如自定义脚本灵活。

### 方案 B: 保持现状（已否决）

**描述**: 继续分别复制多个目录。

**优点**:
- 无变更成本

**缺点**:
- 问题持续存在
- 不利于分发

**否决理由**: 不解决问题，用户反馈明确。
