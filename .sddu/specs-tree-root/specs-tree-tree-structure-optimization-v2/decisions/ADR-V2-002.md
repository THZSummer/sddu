# ADR-V2-002: 缺失字段处理策略（自动填充 vs 报错拒绝）

## 状态
PROPOSED

## 背景
当 state.json 缺少必填字段时（如 version、depth、phaseHistory、dependencies），系统需要决定：
- **自动填充**：使用默认值自动修复，记录警告日志
- **报错拒绝**：返回错误，要求调用方提供完整数据

当前代码中两种策略混用：
- StateLoader.create() 使用自动填充（提供默认值）
- TreeStateValidator.validateNewState() 使用自动修复（修复并记录警告）
- 但缺少统一的错误/警告反馈机制

## 决策
我们决定 **分层处理，自动填充优先**：

### 第一层：StateLoader.create()（写入前）
- **策略**：自动填充所有缺失必填字段
- **行为**：
  - `version`：强制设为 `'v2.1.0'`
  - `depth`：根据 featurePath 自动计算
  - `phaseHistory`：初始化为包含当前阶段记录的数组
  - `dependencies`：初始化为 `{ on: [], blocking: [] }`
  - `files`：初始化为 `{ spec: 'spec.md' }`
  - `childrens`：初始化为空数组 `[]`
  - `createdAt`/`updatedAt`：设为当前时间

### 第二层：TreeStateValidator.validate()（校验时）
- **策略**：校验并自动修复，返回修复结果
- **行为**：
  - 检测缺失字段
  - 可修复的字段自动修复
  - 返回 `autoFixed` 数组记录修复的字段
  - 返回 `warnings` 数组记录警告信息

### 第三层：StateLoader.get()（读取时）
- **策略**：自动修复历史数据（仅内存中）
- **行为**：
  - 检测已知问题（EC-012/013/014）
  - 在内存中修复，不写回文件
  - 记录 WARNING 日志

### 不可修复场景
以下情况返回 `valid: false`，不自动修复：
- `feature` 字段类型错误（非字符串）
- `phase` 字段类型错误（非数字）
- `status` 字段为无效的工作流状态值

## 后果
### 正面
- 用户体验好：不需要手动填写所有字段
- 向后兼容：历史 state.json 可以正常使用
- 可追踪：所有自动修复都有日志记录

### 负面
- 可能掩盖调用方的 Bug（字段缺失本应暴露问题）
- 需要仔细区分"可修复"和"不可修复"场景

### 风险
- 低：自动修复的是有明确默认值的字段，不会产生歧义

## 替代方案
| 方案 | 描述 | 未选择原因 |
|------|------|-----------|
| 报错拒绝 | 缺少必填字段时抛出错误 | 破坏用户体验，历史数据无法使用 |
| 静默填充 | 自动填充但不记录日志 | 问题无法追踪，调试困难 |
