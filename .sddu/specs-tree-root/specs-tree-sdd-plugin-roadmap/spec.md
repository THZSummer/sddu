# SDD Roadmap 规划专家 - 需求规格

## 🎯 基本信息
- **Feature ID**: sdd-plugin-roadmap
- **名称**: SDD Roadmap 规划专家
- **版本**: v1.0.0
- **创建日期**: 2026-04-06
- **Agent**: `@sdd-roadmap`

## 📋 功能目标

创建专门用于**多版本 Roadmap 规划**的 Agent，补充现有 6 阶段工作流的不足。

### 核心能力
1. **Roadmap 规划**: 制定多版本迭代的路线图
2. **Feature 优先级排序**: 使用 RICE 等科学方法评估功能优先级
3. **版本时间表规划**: 基于团队资源、复杂度等因素规划版本发布时间
4. **依赖关系分析**: 分析多模块/多 Feature 间的依赖关系，优化规划顺序
5. **风险评估与建议**: 识别规划中的潜在风险，提出改进建议

## 🎪 需求背景

1. 当前 SDD 工作流专注于单个功能开发的 6 个阶段
2. 缺乏对多版本、多年度整体规划的支持
3. 需要提供版本规划、Roadmap 制定、Feature 优先级排序能力
4. 与 @sdd-2-plan 的区别：单 Feature vs 多版本多 Feature

## 📝 功能需求

### FR-001: 灵活的输入方式
- 支持单个 Feature 输入
- 支持多个 Feature 批量输入
- 支持只有约束条件的输入（如"Q2 上线，2 个人"）
- 支持完全开放的输入（如"接下来做什么好"）
- 支持基于现有文档的规划

### FR-002: 自动信息收集
- 主动询问团队资源（人力、时间）
- 主动询问业务目标（优先级、市场窗口）
- 主动询问技术约束（外部依赖、技术栈）
- 扫描项目上下文获取已有 Feature 状态

### FR-003: RICE 优先级评分
- Reach (触及范围): 预估影响的用户数量或业务范围
- Impact (影响程度): 功能对用户或业务的潜在正面影响程度
- Confidence (可信度): 对 Reach 和 Impact 估算的自信程度
- Effort (工作量): 实现该功能预计所需工作量
- 公式：`RICE Score = (Reach × Impact × Confidence) / Effort`

### FR-004: 依赖关系分析
- 前置依赖分析
- 技术共享分析
- 业务协同分析
- 依赖关系可视化

### FR-005: 多版本时间规划
- 版本划分建议
- 时间预测和工作量分配
- 风险控制（缓冲时间）

### FR-006: 输出格式规范
- 执行摘要放在文档顶部（前 20%）
- 版本总览表
- 功能优先级排序（Top 5）
- 多版本详细规划
- 关键里程碑
- 下一步行动

## ⚠️ 输出边界

### ✅ 允许输出
| 文件 | 说明 |
|------|------|
| `.sdd/ROADMAP.md` | **唯一输出文件**，完整的多版本规划文档 |

### ❌ 禁止输出
| 文件 | 原因 |
|------|------|
| `.sdd/specs-tree-root/specs-tree-[feature]/tasks.md` | 越界，属于任务分解阶段 |
| `.sdd/specs-tree-root/specs-tree-[feature]/plan.md` | 越界，属于技术规划阶段 |
| `.sdd/ROADMAP-EXECUTIVE-SUMMARY.md` | 冗余，执行摘要应放在 ROADMAP.md 顶部 |
