## 🎯 迁移概览

### 背景
OpenCode SDD (Specification-Driven Development) 已升级为 SDDU (Specification-Driven Development Ultimate)，提供更强大的功能和更好的用户体验，同时保持完整向后兼容性。

### 目标
- 保障现有项目零中断切换
- 实现 SDD 到 SDDU 的平滑过渡  
- 为用户提供新旧版本对照及迁移选项

### 迁移策略：**双版本并存 + 渐进式迁移**

## 🛠️ 迁移步骤

```bash

opencode

```

```bash

```

### 任选步骤 (按需要执行)

#### 3A. 继续使用 SDD 旧版 (保守选项)
```bash
# 旧命令继续可用
```

```bash
@sddu-spec "功能Y"              # 更新增强版
```

## 🔄 迁移模式

### 模式 1: 保守兼容模式（推荐用于生产环境）
- **策略**: 继续使用全量 `@sdd-*` 旧命令
- **状态**: 完全兼容，无任何中断
- **优点**: 零风险，无学习成本
- **缺点**: 无法享受 SDDU 新功能

### 模式 2: 混合使用模式（推荐用于新项目）
- **策略**: 新功能使用 `@sddu-*`，核心业务仍用 `@sdd-*`
- **状态**: 完全兼容，渐进采用
- **优点**: 按需尝试新功能，兼容现有业务  
- **缺点**: 命令风格不统一

### 模式 3: 统一升级模式（推荐用于新团队）
- **策略**: 统一使用 `@sddu-*` 全套命令
- **状态**: 兼容旧项目，统一新标准
- **优点**: 充分利用新功能，标准统ー
- **缺点**: 需要团队学习新命令

## 📖 常见问题解答

### Q: 现有的 SDD 项目会中断吗？
**A**: 不会！所有 `@sdd-*` 命令继续保持完成功能，现有项目可以继续正常工作。

### Q: 需要重建 `.sdd/` 目錄嗎？
**A**: 不需要。`.sdd/` 目錄结构保持不变，所有旧文件继续兼容。

### Q: 新舊命令可以混合使用嗎？  
**A**: 可以！同一个项目中可同时使用 `@sdd-*` 和 `@sddu-*`，它们操作相同的数据结构。

### Q: SDDU 有哪些 SDD 没有的機能？
**A**: 
- 7 阶段工作流（新增发现阶段）
- 推荐的新 `@sddu-*` 命令
- 更优化的用户体验

### Q: 将來会棄用 SDD 命令嗎？
**A**: 短期内不會，將提供至少 24 個月的兼容期。長期遷移到 SDDU 是推荐的。

### Q: 如何檢查我的項目是在 SDD 还是 SDDU 模式？
**A**: 查看是否使用 `@sddu-*` 命令或者項目中有使用 SDDU 特有的發現功能。

## 🔧 技术细节

### 系统兼容性
- SDD 和 SDDU 共享相同的底层数据结构（`.sdd/` 目錄）
- 相同的狀態管理和檔案格式
- 共享相冋的插件架構

### 版本識別策略
```
SDDU 1.4.0 = SDD 1.1.1 的升級版本
  - 新功能: @sddu-* 系列命令
  - 兼容性: @sdd-* 系列命令完全保持
  - 数据層: 同樣的 .sdd/ 資料結構
```

### 遾難恢復計劃
如果遭遇問題，可使用現有的 `@sdd-*` 命令繼續作業，確保業務不中斷。
