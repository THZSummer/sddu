## ✨ 核心功能

### 1. 6 阶段 SDD 工作流

完整的 Specification-Driven Development 流程：

```
1.spec → 2.plan → 3.tasks → 4.build → 5.review → 6.validate
```

| 阶段 | Agent | 输入 | 输出 |
|------|-------|------|------|
| 1/6 | @sdd-spec | 需求描述 | spec.md |
| 2/6 | @sdd-plan | spec.md | plan.md + ADR |
| 3/6 | @sdd-tasks | plan.md | tasks.md |
| 4/6 | @sdd-build | tasks.md | 代码实现 |
| 5/6 | @sdd-review | 代码 + 文档 | 审查报告 |
| 6/6 | @sdd-validate | 完整 feature | 验证报告 |

### 2. Agent 系统 (8 个)

| Agent | 用途 | 模式 |
|-------|------|------|
| `@sdd` | 智能路由助手 | subagent |
| `@sdd-help` | 帮助助手 | subagent |
| `@sdd-spec` | 规范编写专家 | subagent |
| `@sdd-plan` | 技术规划专家 | subagent |
| `@sdd-tasks` | 任务分解专家 | subagent |
| `@sdd-build` | 任务实现专家 | subagent |
| `@sdd-review` | 代码审查专家 | subagent |
| `@sdd-validate` | 验证专家 | subagent |

### 3. 状态机管理

- Feature 状态追踪
- 自动进度管理
- 状态持久化 (JSON)

### 4. 插件工具

| 工具 | 说明 |
|------|------|
| `sdd_init` | 初始化 SDD 工作流 |
| `sdd_specify` | 创建 Feature Specification |
| `sdd_status` | 查看 SDD 状态 |

### 5. 模板系统

- Handlebars 模板引擎
- 8 个 Agent 模板
- 规范模板 (spec.md.hbs)

## 🚀 安装方法

### 项目级安装（推荐）

```bash
# 1. 构建插件
cd opencode-sdd-plugin
npm install
npm run build

# 2. 复制到项目
cp -r dist/* <your-project>/.opencode/plugins/sdd/

# 3. 配置 opencode.json
{
  "plugins": [
    {
      "name": "opencode-sdd-plugin",
      "path": "./.opencode/plugins/sdd"
    }
  ]
}

# 4. 重启 OpenCode
opencode
```

## 📊 已知限制

### v1.0.0 限制

1. **状态机未完全集成**
   - Agent 模板没有实际调用状态机
   - 状态更新需要手动执行

2. **权限配置不一致**
   - 部分 Agent 缺少 frontmatter
   - 权限定义不统一

3. **模板格式不统一**
   - 部分 Agent 缺少示例对话
   - 输出格式不一致

4. **缺少前后端对齐检查**
   - 没有专门的对齐检查 Agent
   - 前后端 spec 不一致问题需手动发现

> 这些限制将在后续版本中修复（见 Phase 计划）
