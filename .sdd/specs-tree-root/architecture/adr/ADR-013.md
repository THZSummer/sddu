## 1. 背景

### 1.1 问题陈述

当前 SDD 插件的 Agent 注册是静态硬编码的：

**现状** (`src/agents/sdd-agents.ts`):
```typescript
const agents = [
  {
    name: 'sdd-spec',
    description: 'SDD 规范编写专家 (阶段 1/6)',
    mode: 'subagent',
    promptFile: '.opencode/agents/sdd-spec.md'
  },
  // ... 硬编码列表
];
```

**问题**:
1. **扩展性差**: 添加新 Agent 需要修改核心代码
2. **不支持插件化**: 第三方无法扩展 Agent
3. **配置分散**: Agent prompt 路径硬编码，难以统一管理
4. **版本更新困难**: 更新 Agent 需要重新发布插件

### 1.2 用户反馈

| 反馈 | 日期 | 影响 |
|------|------|------|
| "sdd 插件的信息在哪里，没有 plugin 目录吗" | 2026-04-05 | 插件结构不清晰 |
| "Agent 配置放在 `.opencode/agents/` 导致配置分散" | 2026-04-05 | 不符合插件标准结构 |

### 1.3 决策范围

本 ADR 决定：
- 实现 `AgentRegistry` 类支持动态注册
- Agent prompt 放在插件目录内 (`.opencode/plugins/sdd/agents/`)
- 支持静态注册和动态加载两种方式
- 保持向后兼容

---

## 2. 决策

### 2.1 方案选择

我们决定采用 **混合注册机制** 方案：

**创建 `AgentRegistry` 类，支持静态注册和从插件目录动态加载**。

### 2.2 设计原则

1. **双模式支持**: 静态注册 + 动态加载
2. **插件自包含**: Agent prompt 放在插件目录内
3. **优先级机制**: 静态注册优先级高于动态加载
4. **静默失败**: 动态加载失败不影响静态 Agent

### 2.3 目录结构

```
sddu/ (本仓库)
├── src/templates/agents/       # Agent prompt 模板源码
│   ├── sdd.md
│   ├── spec.md
│   ├── plan.md
│   └── ...
└── ...

用户项目/ (安装后)
├── .opencode/
│   └── plugins/
│       └── sdd/
│           ├── index.js        # 插件入口
│           └── agents/         # Agent prompt
│               ├── sdd.md
│               ├── spec.md
│               ├── plan.md
│               └── ...
```

### 2.4 实现细节

#### 2.4.1 AgentRegistry 类

```typescript
import type { AgentMetadata } from '../types';

export interface RegisteredAgent extends AgentMetadata {
  registeredAt: string;
  source: 'static' | 'dynamic';
}

export class AgentRegistry {
  private agents: Map<string, RegisteredAgent> = new Map();

  /**
   * 注册单个 Agent
   */
  register(agent: AgentMetadata, source: 'static' | 'dynamic' = 'static'): void {
    if (this.agents.has(agent.name)) {
      console.warn(`Agent 已存在：${agent.name}，将被覆盖`);
    }

    this.agents.set(agent.name, {
      ...agent,
      registeredAt: new Date().toISOString(),
      source
    });
  }

  /**
   * 批量注册 Agents
   */
  registerMany(agents: AgentMetadata[], source?: 'static' | 'dynamic'): void {
    for (const agent of agents) {
      this.register(agent, source);
    }
  }

  /**
   * 获取单个 Agent
   */
  get(name: string): RegisteredAgent | undefined {
    return this.agents.get(name);
  }

  /**
   * 获取所有 Agents
   */
  getAll(): RegisteredAgent[] {
    return Array.from(this.agents.values());
  }

  /**
   * 按阶段过滤 Agents
   */
  getByCategory(category: AgentMetadata['category']): RegisteredAgent[] {
    if (!category) return this.getAll();
    return this.getAll().filter(agent => agent.category === category);
  }

  /**
   * 从插件目录动态加载 Agents
   */
  async loadFromDirectory(dirPath: string): Promise<number> {
    const fs = await import('fs/promises');
    const path = await import('path');

    let count = 0;

    try {
      const files = await fs.readdir(dirPath);
      const mdFiles = files.filter(f => f.endsWith('.md') || f.endsWith('.md.hbs'));

      for (const file of mdFiles) {
        const agentName = file.replace('.md.hbs', '').replace('.md', '');
        const metadata: AgentMetadata = {
          name: `sdd-${agentName}`,
          description: `SDD ${agentName} Agent`,
          mode: 'subagent',
          promptFile: path.join(dirPath, file),
          category: this.inferCategory(agentName)
        };

        this.register(metadata, 'dynamic');
        count++;
      }
    } catch (error) {
      console.error('加载 Agents 失败:', error);
    }

    return count;
  }

  private inferCategory(name: string): AgentMetadata['category'] {
    const categoryMap: Record<string, AgentMetadata['category']> = {
      'spec': 'spec',
      'plan': 'plan',
      'tasks': 'tasks',
      'build': 'build',
      'review': 'review',
      'validate': 'validate',
      'roadmap': 'utility',
      'docs': 'utility',
      'help': 'utility',
      'sdd': 'utility'
    };
    return categoryMap[name] || 'utility';
  }
}

// 单例实例
export const agentRegistry = new AgentRegistry();
```

#### 2.4.2 修改 sdd-agents.ts

```typescript
import { agentRegistry } from './registry';
import type { AgentMetadata } from '../types';

export async function registerAgents(context: any) {
  // 定义静态 Agents（向后兼容）
  const staticAgents: AgentMetadata[] = [
    {
      name: 'sdd',
      description: 'SDD 工作流智能入口 - 自动路由到正确阶段',
      mode: 'subagent',
      promptFile: '.opencode/plugins/sdd/agents/sdd.md',
      category: 'utility'
    },
    {
      name: 'sdd-spec',
      description: 'SDD 规范编写专家 (阶段 1/6)',
      mode: 'subagent',
      promptFile: '.opencode/plugins/sdd/agents/spec.md',
      category: 'spec'
    },
    // ... 其他 Agents
  ];

  // 注册静态 Agents
  agentRegistry.registerMany(staticAgents, 'static');

  // 动态加载插件目录下的 Agents
  const pluginsDir = path.join(context.directory, '.opencode', 'plugins', 'sdd', 'agents');
  const loadedCount = await agentRegistry.loadFromDirectory(pluginsDir);
  
  console.log(`✅ 已注册 ${agentRegistry.getAll().length} 个 SDD Agents (${loadedCount} 动态加载)`);

  return {
    agents: agentRegistry.getAll(),
    updateStateForAgentCall: createUpdateStateFunction()
  };
}
```

### 2.5 安装流程

```
构建流程:
1. tsc 编译 TypeScript 到 dist/
2. src/templates/agents/*.md 复制到 dist/templates/agents/
3. package.cjs 打包到 dist/sdd/ 和 dist/sdd.zip

安装流程:
1. install.sh/install.ps1 复制 dist/ 到 .opencode/plugins/sdd/
2. dist/templates/agents/ → .opencode/plugins/sdd/agents/

1. 插件加载时调用 registerAgents()
2. 静态 Agents 注册
3. 从 .opencode/plugins/sdd/agents/ 动态加载
```

## 3. 后果

### 3.1 正面影响

| 影响 | 说明 |
|------|------|
| ✅ 支持插件化扩展 | 第三方可添加自定义 Agent |
| ✅ 配置集中 | Agent prompt 在插件目录内 |
| ✅ 符合插件标准 | `.opencode/plugins/sdd/` 结构 |
| ✅ 易于更新 | 更新 prompt 无需修改核心代码 |
| ✅ 向后兼容 | 静态注册仍然有效 |

### 3.2 负面影响

| 影响 | 说明 | 缓解措施 |
|------|------|----------|
| ⚠️ 增加复杂度 | 需要维护注册表逻辑 | 代码审查确保质量 |
| ⚠️ 动态加载可能失败 | 目录不存在或权限问题 | 静默跳过，不影响静态 Agent |
| ⚠️ 安装脚本需要更新 | 适配新目录结构 | 提供降级模式 |

### 3.3 技术影响

- **编译时间**: 无明显影响
- **运行时性能**: 轻微增加（动态加载），异步不阻塞
- **包大小**: 增加约 5KB（压缩后）
- **向后兼容**: 完全兼容

## 4. 替代方案

### 方案 A: 完全动态加载（已否决）

**描述**: 只支持从目录动态加载，移除静态注册。

**优点**:
- 最灵活
- 完全插件化

**缺点**:
- 依赖文件系统
- 启动时可能失败
- 无兜底机制

**否决理由**: 风险高，需要静态兜底。

### 方案 B: 保持现状（已否决）

**描述**: 继续硬编码 Agent 列表。

**优点**:
- 无变更成本

**缺点**:
- 扩展性差
- 不符合用户反馈

**否决理由**: 不解决问题，用户反馈明确。
