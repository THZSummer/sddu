# SDD Roadmap Agent 规格说明

## 基本信息
- **Feature ID**: sdd-plugin-roadmap
- **名称**: SDD Roadmap Agent
- **版本**: 1.0.0
- **作者**: SDD 团队
- **创建日期**: 2026-03-30

## 功能目标
创建专门用于**多版本 Roadmap 规划**的 Agent，补充现有 6 阶段工作流的不足。

## 需求背景
1. 当前 SDD 工作流专注于单个功能开发的 6 个阶段
2. 缺乏对多版本、多年度整体规划的支持
3. 需要提供版本规划、Roadmap 制定、Feature 优先级排序能力
4. 与 @sdd-2-plan 的区别：单 Feature vs 多版本多 Feature

## 功能特性
1. **Roadmap 规划**: 协助制定多个版本迭代的路线图
2. **Feature 优先级排序**: 使用 RICE 等方法评估功能优先级
3. **版本时间表规划**: 根据团队资源、复杂度等因素规划版本发布时间
4. **依赖关系分析**: 分析多模块间的依赖关系并优化规划顺序
5. **风险评估**: 识别规划中的潜在风险和问题

## 与其他组件关系
- **扩展 @sdd-2-plan**: 在单功能详细规划基础上支持多功能、多版本规划
- **配合 @sdd-tasks**: 生成高层次的功能任务分解建议
- **依赖 @sdd-spec**: 获取待规划功能的详细需求规格

## 技术要求
1. 遵循 Markdown Agent 格式规范
2. 与现有 14 个 Agent 风格保持一致
3. 支持详细的 Roadmap 输出格式
4. 集成优先级评估算法(RICE评分等)

## 成功指标
- [ ] 可以生成清晰的 Roadmap 计划表
- [ ] 可以合理评估功能之间的优先级
- [ ] 支持多版本时间安排
- [ ] 提供详细的依赖分析结果
- [ ] 与现有 SDD 工作流无缝集成
