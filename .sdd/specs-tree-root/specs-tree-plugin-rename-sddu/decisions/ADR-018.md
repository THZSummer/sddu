# ADR-018: 配置文件更新方案 - opencode.json 等文件处理

**Feature**: FR-SDDU-RENAME-001 插件改名 SDDU  
**状态**: PROPOSED  
**创建日期**: 2026-04-06  
**决策者**: SDD Plan Agent  

---

## 1. 背景

配置文件改名是本次改名的核心环节，涉及多个关键文件：

### 配置文件清单

| 文件 | 类型 | 影响范围 |
|------|------|---------|
| `/package.json` | 包配置 | 全局 |
| `/opencode.json` | 插件配置 | 全局 |
| `/.opencode/plugins/sdd/opencode.json` | 插件内配置 | 插件 |
| `/.opencode/plugins/sdd/package.json` | 插件包配置 | 插件 |
| `/src/templates/agents/*.hbs` | Prompt 模板 | Agent 行为 |

### 关键变更点

1. **package.json**:
   - `name`: `opencode-sdd-plugin` → `opencode-sddu-plugin`
   - `description`: 更新品牌描述
   - `scripts`: 所有 `sdd-*` → `sddu-*`
   - `keywords`: 添加 `sddu`
   - `files`: `dist/sdd/**/*` → `dist/sddu/**/*`

2. **opencode.json**:
   - `plugin`: 数组引用更新
   - `agent`: 36 个定义（18 旧 + 18 新）

3. **模板文件**:
   - 文件名：`sdd*.hbs` → `sddu*.hbs`
   - 内容：`@sdd-*` → `@sddu-*`

---

## 2. 方案对比

### 方案 A: 直接替换（推荐）

使用查找替换工具批量更新所有配置文件。

**工具链**:
- `ripgrep` 或 `grep` 查找所有引用
- 脚本化替换确保一致性
- 手动审查关键文件

**优点**:
- 执行快速
- 可脚本化
- 易于验证

**缺点**:
- 可能误替换（如注释中的说明）
- 需要仔细审查

### 方案 B: 手动逐文件修改

逐文件打开并手动修改。

**优点**:
- 精确控制每处变更
- 可同步更新相关注释

**缺点**:
- 耗时
- 容易遗漏
- 人为错误风险高

### 方案 C: 混合方式

核心配置手动修改，模板和文档批量替换。

**优点**:
- 关键文件精确控制
- 大量文件高效处理

**缺点**:
- 需要区分"核心"和"非核心"
- 流程复杂

---

## 3. 决策

**我们决定**: 采用 **方案 C - 混合方式**

### 具体实施

#### 3.1 核心配置（手动修改）

以下文件手动精确修改：

1. **`/package.json`**:
   ```json
   {
     "name": "opencode-sddu-plugin",
     "description": "Specification-Driven Development Unified plugin for OpenCode...",
     "keywords": ["opencode", "plugin", "sddu", "specification-driven"],
     "files": ["dist/sddu/**/*"],
     "scripts": {
       "sddu-spec": "node ./dist/commands/sddu-spec.js",
       "sddu-plan": "node ./dist/commands/sddu-plan.js",
       // ... 保留旧的 sdd-* 脚本作为别名
       "sdd-spec": "npm run sddu-spec",
       "sdd-plan": "npm run sddu-plan"
     }
   }
   ```

2. **`/opencode.json`**:
   - 手动添加 36 个 agent 定义
   - 确保新旧 agent 指向正确的 prompt 文件
   - 添加注释说明双定义原因

3. **`/.opencode/plugins/sdd/opencode.json`**:
   - 同上

#### 3.2 模板文件（批量替换）

使用脚本批量处理模板文件：

```bash
# 重命名文件
for file in src/templates/agents/sdd*.hbs; do
  mv "$file" "${file/sdd/sddu}"
done

# 替换内容
find src/templates/agents/ -name "*.hbs" -exec sed -i 's/@sdd-/ @sddu-/g' {} \;
find src/templates/agents/ -name "*.hbs" -exec sed -i 's/\.sdd\//.sddu\//g' {} \;
```

#### 3.3 源代码文件（查找 + 手动审查）

1. 使用 `grep` 查找所有 `sdd`/`SDD` 引用
2. 分类为：
   - 必须修改（类名、常量名）
   - 建议修改（注释、日志）
   - 保留（历史引用、CHANGELOG）
3. 逐类处理

#### 3.4 文档文件（批量 + 审查）

1. 批量替换明显的品牌引用
2. 手动审查：
   - API 文档
   - 迁移指南
   - FAQ

## 4. 配置文件变更清单

### 4.1 package.json (根目录)

| 字段 | 旧值 | 新值 | 处理方式 |
|------|------|------|---------|
| `name` | `opencode-sdd-plugin` | `opencode-sddu-plugin` | 手动 |
| `description` | `Specification-Driven Development plugin...` | `Specification-Driven Development Unified plugin...` | 手动 |
| `keywords` | `["opencode", "plugin", "sdd"]` | `["opencode", "plugin", "sddu", "specification-driven"]` | 手动 |
| `files` | `dist/sdd/**/*` | `dist/sddu/**/*` | 手动 |
| `scripts.sdd-*` | `node ./dist/commands/sdd-*.js` | 保留作为别名，指向新脚本 | 手动 |
| `scripts.sddu-*` | - | `node ./dist/commands/sddu-*.js` | 手动添加 |

### 4.2 opencode.json (根目录)

| 配置项 | 变更 | 处理方式 |
|--------|------|---------|
| `plugin` | `["opencode-sdd-plugin"]` → `["opencode-sddu-plugin"]` | 手动 |
| `agent.sdd` | 添加，deprecated | 手动 |
| `agent.sddu` | 添加，主定义 | 手动 |
| `agent.sdd-*` | 添加 16 个 deprecated 定义 | 手动 |
| `agent.sddu-*` | 添加 16 个主定义 | 手动 |

### 4.3 模板文件

| 操作 | 数量 | 处理方式 |
|------|------|---------|
| 文件重命名 | 11 个 | 脚本批量 |
| 内容替换 | 11 个 | 脚本批量 + 手动审查 |

### 4.4 源代码

| 文件 | 变更 | 处理方式 |
|------|------|---------|
| `src/index.ts` | `SDDPlugin` → `SDDUPlugin`, `sdd-plugin` → `sddu-plugin` | 手动 |
| `src/agents/sdd-agents.ts` | 可选重命名为 `sddu-agents.ts` | 手动 |
| `src/errors.ts` | `SddError` → `SdduError`, `[SDD-` → `[SDDU-` | 手动 |

## 5. 后果

### 正面影响

✅ **命名统一**: 所有配置使用一致的新品牌  
✅ **向后兼容**: npm scripts 保留旧名称作为别名  
✅ **可追溯**: 使用 `git mv` 保持文件历史  
✅ **可验证**: 配置变更清晰，易于测试

### 负面影响

❌ **配置文件膨胀**: opencode.json 增加约 2 倍内容  
❌ **构建路径变更**: dist 目录结构需要更新  
❌ **安装脚本更新**: install.sh/ps1 需要适配新路径

### 缓解措施

1. **配置注释**: 在 opencode.json 中添加注释说明结构
2. **构建验证**: CI 中增加配置验证步骤
3. **安装测试**: 发布前验证安装脚本
