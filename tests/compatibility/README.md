# SDD 向后兼容性测试

## 概述

此目录包含了 SDD 项目的向后兼容性测试方案，旨在确保新版本（v1.2.11）能够无缝兼容旧版本项目，并提供了安全的状态格式迁移机制。

## 迁移目标

从旧版本 (v1.1.1) 迁移至统一的状态格式 (v1.2.11)，主要变化包括：

### 删除字段
- `mode`: 由目录结构自动识别
- `subFeatures`: 通过扫描同级目录自动发现

### 新增字段  
- `version`: 版本标识 (`1.2.11`)

### 保留字段  
- `feature`, `name`, `status`, `phase`, `files`, `dependencies`, `assignee`, `createdAt`, `updatedAt`

## 测试场景

### 1. 数据迁徙测试
验证 v1.1.1 格式的数据能正确迁移为 v1.2.11 格式

### 2. 功能兼容测试 
确认旧项目的各项功能在新模式下依旧可用

### 3. 回滚机制测试
确保迁移过程中有可靠备份机制

### 4. Agent 兼容测试
确保各核心 Agent 可处理不同版本格式的 State 信息

## 验收标准

- [x] v1.1.1 state.json 可自动升级为 v1.2.11
- [x] 升级后移除 mode 和 subFeatures 字段 
- [x] 单模块项目无需迁移可正常工作
- [x] 核心 Agent（@sdd-spec/plan/tasks/build）正常调用
- [x] 迁移后备份文件存在且可回滚
- [x] 旧 `.specs/` 结构项目可正常加载
- [x] 兼容性测试报告完整

## 设计特色

1. **自动检测** - 系统自动识别旧版本格式并提示可用迁移
2. **安全备份** - 迁移前自动创建备份文件
3. **渐进兼容** - 未强制迁移，允许逐步升级
4. **无缝体验** - 用户不需要手动执行迁移操作

## 版本演进轨迹

```
v1.1.1: 包含 mode 和 subFeatures 字段的传统格式
   ↓
v1.2.11: 简化的统一格式，mode 和 subFeatures 由目录结构决定
```

## 注意事项

- 旧的 `.specs/` 结构仍然支持，但推荐迁移到新的 `.sdd/.specs/` 容器化结构
- 自动备份存放在 `.sdd/.backups/` 目录下
- 迁移不可逆（可通过备份恢复）
